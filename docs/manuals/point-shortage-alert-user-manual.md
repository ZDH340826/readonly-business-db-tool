# 点位缺料报警系统使用说明书

## 1. 软件定位

本软件用于现场运维人员按区域、物料组监控点位是否缺少货架。软件只查询数据库，不提供 SQL 编辑器，不提供表数据修改界面，不向业务数据库写入任何数据。

核心场景：

- 每个运维人员只维护自己负责区域的点位组。
- 一个点位组通常包含 1 个使用位和多个备用位。
- 系统每 60 秒查询一次数据库。
- 运维人员可配置“持续缺料多少分钟后报警”。
- 报警弹窗必须手动点击“已关注”才会关闭。
- 每次检测和报警事件都会写入本地日志文件。

## 3. 首次运行

1. 双击 `ShelfPointMonitor.bat`。
2. 进入“连接管理”。
3. 选择或新建连接。
4. 输入数据库密码。
5. 点击“测试连接并使用”。
6. 进入“点位缺料报警”。
7. 配置点位组和报警规则。
8. 点击“保存配置”。
9. 点击“立即检测”验证。
10. 点击“开始监控”进入自动检测。

密码只保存在本次运行内存中，不写入配置文件。

## 4. 连接管理

字段说明：

| 字段 | 含义 |
|---|---|
| 连接名称 | 本地连接别名，例如“现场数据库” |
| 数据库类型 | `postgres` 或本地测试库 `h2` |
| 服务器地址/IP | 数据库服务器地址，由现场运维本机填写 |
| 端口 | 现场默认 `2345` |
| 数据库名 | 现场默认 `cms_web` |
| 数据库空间/Schema | 现场默认 `public` |
| 用户名 | 数据库用户名，由现场运维本机填写 |
| 密码 | 数据库密码，仅本次运行使用，不保存 |
| SSL模式 | PostgreSQL SSL 模式，现场默认 `disable` |
| 本地测试库路径 | H2 本地测试库文件路径 |

建议生产环境使用数据库只读账号。

连接配置保存到本机运行目录 `data/connections.properties`。软件不会把密码写入该文件。

## 5. 点位组配置

一个点位组用于描述同一种物料的一组点位。

推荐结构：

```text
区域 A
└─ 后围板组
   ├─ 使用位：USE_POINT_001
   ├─ 备用位：BACKUP_POINT_001
   ├─ 备用位：BACKUP_POINT_002
   ├─ 备用位：BACKUP_POINT_003
   └─ 备用位：BACKUP_POINT_004
```

字段说明：

| 字段 | 含义 |
|---|---|
| 组ID | 本地点位组唯一标识 |
| 区域 | 现场区域名 |
| 组名 | 点位组名称 |
| 物料 | 该组对应物料 |
| 启用 | 是否监控该组 |
| 启用规则 | 是否启用该组报警规则 |
| 使用位无货架 | 规则是否要求使用位为空 |
| 最少备用位有料 | 备用位至少需要多少个有货架 |
| 持续分钟 | 规则连续满足多少分钟后报警 |

## 6. 报警规则

当前规则模型：

```text
使用位无货架
并且
备用位有货架数量 < 最少备用位有料
并且
上述状态持续达到配置分钟数
```

示例：

```text
使用位：无货架
备用位总数：4
备用位有货架：2
最少备用位有料：3
持续分钟：5
```

含义：

- 如果使用位短暂无货架，但备用位仍满足规则，不报警。
- 如果使用位无货架且备用位不足，并连续 5 次检测都成立，系统报警。
- 系统固定每 60 秒检测一次，因此持续 5 次约等于 5 分钟。

## 7. 报警处理

报警后会弹出窗口，显示区域、点位组、物料、备用位有料数量和持续缺料时间。

处理流程：

1. 现场确认点位状态。
2. 处理物料或调度问题。
3. 在报警弹窗点击“已关注”。
4. 如果缺料仍未恢复，系统不重复弹出同一轮已关注报警。
5. 点位组恢复正常后，后续再次触发规则会重新报警。

## 8. 日志

日志目录：`logs/`

| 文件 | 内容 |
|---|---|
| `monitor.log` | 界面运行日志 |
| `check-log.csv` | 每次检测快照 |
| `event-log.csv` | 报警、确认、恢复事件 |

`check-log.csv` 主要字段：

| 字段 | 含义 |
|---|---|
| checked_at | 检测时间 |
| group_id | 点位组 ID |
| status | 当前状态 |
| use_point_empty | 使用位是否无货架 |
| backup_total | 备用位总数 |
| backup_available | 有货架备用位数量 |
| backup_missing | 无货架备用位数量 |
| continuous_minutes | 连续满足报警规则分钟数 |

`event-log.csv` 事件类型：

| 事件 | 含义 |
|---|---|
| ALERT_OPEN | 达到报警条件并弹窗 |
| ACKNOWLEDGED | 用户点击已关注 |
| RECOVERED | 点位组恢复正常 |

## 9. 本地测试库

不连接真实数据库时，可使用内置 H2 本地测试库。

脚本：

| 脚本 | 用途 |
|---|---|
| `LocalTest_Reset.bat` | 重置测试库 |
| `LocalTest_Normal.bat` | 切换为正常场景 |
| `LocalTest_MissingUse.bat` | 切换为缺使用位场景 |
| `LocalTest_MissingBackup.bat` | 切换为缺备用位场景 |

本地测试库默认包含 1 个使用位和 4 个备用位，使用示例点位编码，不包含现场真实点位信息。

## 10. 只读边界

软件层面：

- 不提供 SQL 编辑器。
- 不提供增删改按钮。
- 数据库浏览器只做 metadata 查询和固定 SELECT 预览。
- 点位监控只执行固定 SELECT 查询。
- 日志只写本机 `logs/` 目录。

数据库层面建议：

- 创建只读账号。
- 只授权必要 schema/table 的 SELECT。
- 不要使用 DBA 或业务写入账号运行本软件。

## 11. 常见问题

### 点击检测提示未连接

先进入“连接管理”，点击“测试连接并使用”。

### 报警没有立即出现

检查点位组的“持续分钟”。如果设置为 5，系统需要连续 5 次检测都满足规则才报警。

### 修改规则后没有生效

点击“保存配置”。如果自动监控已启动，建议停止后重新开始。

### 日志文件无法写入

检查 `logs/` 目录权限，或确认 CSV 文件没有被其他软件独占打开。

### 担心误改数据库

使用只读数据库账号，并确认数据库权限只包含 SELECT。

### 哪些信息不能提交到 GitHub

真实 IP、用户名、密码、生产点位组、真实点位清单、现场导出数据都只能保存在本机运行目录或现场内部文档中，不要提交到公开仓库。
