# 现场运维交付手册

适用版本：`0.5.0-rc.1`。本手册面向 Windows 现场运维和验收人员。所有示例均为公开占位信息；真实主机、账号、点位和导出数据只能保存在现场运行目录。

## 1. 软件用途

工具以只读方式查看业务数据库，并按区域、物料组监控使用位和备用位的缺料风险。它不是通用数据库客户端，不提供 SQL 编辑或数据修改能力。

## 2. 安全边界

生产路径只执行参数化 `SELECT` 与 JDBC 元数据读取。密码只在当前运行内存中使用，不保存。现场必须分配数据库只读账号，并遵守本单位的网络、账号和变更管理要求。

## 3. 交付包核对

确认 ZIP 名为 `ReadonlyBusinessDbTool-v0.5.0-rc.1.zip`，并按同目录 `SHA256SUMS.txt` 复核哈希。解压后必须包含 `ShelfPointMonitor.jar`、`runtime`、`lib`、`data`、`logs`、`diagnostics`、`VERSION`、三个中文脚本和三份中文文档。

## 4. 解压与目录权限

把整个包解压到普通用户可读写目录，路径允许中文和空格。不要放入需要管理员权限的系统目录，不要只复制 JAR。`data`、`logs`、`diagnostics` 必须可写。

## 5. 执行现场部署检查

双击或在终端运行 `现场部署检查.bat`。每项应显示 `[通过]`；任一 `[失败]` 都必须先处理。报告保存在 `diagnostics/preflight-latest.txt`。该检查只使用包内本地 H2，不连接 PostgreSQL。

## 6. 启动和关闭

运行 `启动工具.bat`。正常关闭请使用窗口关闭按钮，程序会停止监控并关闭后台执行器。不要在配置保存或诊断导出过程中强制结束进程。

## 7. 八页工作台

左侧导航依次为“监控总览”“点位组管理”“报警中心”“连接管理”“数据查询”“数据源浏览器”“日志与系统”“系统设置”。顶部显示连接、监控、上次检测和下次检测；底部显示当前操作结果和版本。

## 8. 本地测试库验证

首次验收先在“连接管理”选择“本地测试库”，点击“设为当前连接”和“测试连接”。再到“监控总览”点击“立即检测”，确认样例组能显示点位状态。该步骤不访问生产数据库。

## 9. 新建现场连接

进入“连接管理”，点击“新建连接”，填写连接名称、数据库类型、服务器地址、端口、数据库名、Schema、用户名和 SSL 模式，然后点击“保存连接”。密码不属于保存字段，禁止写入名称或备注。

## 10. 测试并启用连接

在“连接管理”输入本次运行密码，点击“测试连接”。通过后点击“设为当前连接”。现场待确认：网络、认证、SSL、schema 权限和只读账号对固定查询/JDBC metadata 的授权。

## 11. 配置点位组

进入“点位组管理”，使用“新增组”“新增点位”建立区域、物料组、使用位和备用位。设置检测周期、持续时间、备用位最小可用数量及是否参与判断。点击“验证配置”，通过后点击“保存配置”。

## 12. 点位组校验规则

启用组必须至少有一个启用的使用位；同组点位编码不得重复；备用位阈值不得超过启用备用位数量；检测周期和持续时间必须在界面允许范围内；组 ID 必须全局唯一。现场真实编码不得提交回公开仓库。

## 13. 开始与停止监控

在“监控总览”确认当前连接和组配置后点击“开始监控”。需要暂停时点击“停止监控”。停止后旧任务结果会被丢弃，旧密码快照会清零；再次开始会创建新会话。

## 14. 立即检测

“监控总览”的“立即检测”使用点击时的连接、密码和点位组快照。它不会在 Swing 界面线程中访问数据库。检测结果显示在状态表和详情区。

## 15. 状态含义

- “正常”：规则未匹配。
- “观察中”：缺料条件已出现，但尚未达到持续时间。
- “报警”：缺料条件持续达到阈值。
- “已关注”：运维已确认本轮报警，恢复前不重复弹出。
- “已恢复”：真实业务报警后规则恢复正常。
- “查询失败，数据不可用”：本次数据不可用，不解释为缺料、正常或已恢复。

## 16. 报警处理

进入“报警中心”，可按活跃报警、已关注、观察中、查询失败和已恢复筛选。使用“查看点位详情”“立即检测该组”核对情况；确认已处理后点击“标记已关注”。目标按组 ID 识别，不依赖可能重名的显示名称。

## 17. 查询失败与恢复

查询失败会清除旧缺料连续计时并关闭同组旧报警窗口，但不会生成缺料报警。连接恢复后，如果仍缺料，会从零重新计时；如果已有料，则回到正常。先检查“连接管理”和“日志与系统”，不要把查询失败当成现场补料结论。

## 18. 数据查询与导出

“数据查询”只提供结构化条件。点击“执行只读查询”，用“上一页”“下一页”翻页，点击“导出当前结果”只导出当前内存页。CSV 为 UTF-8，已处理引号/换行和电子表格公式注入；导出后仍应按现场数据制度保管。

## 19. 数据源浏览器

在“数据源浏览器”依次点击“刷新 Schema”“加载表/视图”，选择对象后点击“预览前100行”。页面只读，标识符受白名单限制，没有 SQL 输入或写入操作。

## 20. 日志、自检与健康状态

“日志与系统”显示调度器、连接、检测、配置、日志目录和自检健康状态。可用“刷新日志”“打开日志目录”和“执行自检”。页面最多展示最近 1000 条事件、1000 条检测和 200 行运行日志，并在显示前脱敏。

## 21. 系统设置与诊断包

“系统设置”只包含实际生效的八项设置；使用“保存设置”“恢复默认”“重新加载配置”。只读访问、敏感信息脱敏、无 SQL 编辑和禁止密码持久化是不可关闭安全项。需要支持时，在“日志与系统”点击“导出诊断包”，或运行 `生成诊断包.bat`；只提交生成的 ZIP，不提交 `data` 目录。

## 22. 备份、升级、回滚与现场待确认

升级前停止监控并关闭程序，备份旧版本的本地配置与必要日志，新版本并排解压后先运行预检和本地 H2 验证。出现阻断问题按 `回滚说明.md` 恢复旧目录。真实 PostgreSQL 连通性、权限、字段语义、真实点位规则、长时间稳定性和目标 Windows/DPI 均为“现场待确认”，不得用离线测试结论代替。
