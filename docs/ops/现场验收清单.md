## 1. 发布包完整性

## 2. 内嵌 Java

- [ ] 临时移除系统 Java 后，`启动工具.bat --self-test` 仍返回 0；不需要管理员权限或联网。

## 3. 离线部署检查

- [ ] `现场部署检查.bat` 返回 0，全部项目为 `[通过]`，报告写入 `diagnostics\preflight-latest.txt`。

## 4. H2 演示模式

- [ ] “连接管理”选择本地测试库并点击“测试连接”“设为当前连接”；“监控总览”点击“立即检测”后显示公开样例状态。

## 5. 只读账号连接（现场待确认）

- [ ] 使用专用 PostgreSQL 只读账号“测试连接”通过；未向工具或文档保存账号凭据。

## 6. 数据库当前用户确认（现场待确认）

- [ ] DBA 在现场受控工具中确认实际数据库当前用户为批准的只读角色，并确认无写入、DDL 或授权权限；结果未写入本模板。

## 7. Schema 和字段映射（现场待确认）

- [ ] DBA/业务负责人确认目标 Schema、固定表、八个字段、状态值、锁定值、时区和数据新鲜度与发布说明一致。

## 8. 有料验证（现场待确认）

- [ ] 选择经业务确认的有料场景执行“立即检测”，点位和组状态显示“有料/正常”，未触发缺料报警。

## 9. 无料验证（现场待确认）

- [ ] 选择经业务确认的无料场景，状态先进入“观察中”，点位明细和备用位可用数与现场事实一致。

## 10. 持续报警验证（现场待确认）

- [ ] 缺料条件按真实经过时间达到配置阈值后进入“活跃报警”，未提前触发且未执行数据库写入。

## 11. 已关注验证（现场待确认）

- [ ] 点击“标记已关注”后同次短缺不重复弹窗，状态明确显示“已关注”。

## 12. 恢复验证（现场待确认）

- [ ] 真实补料后进入业务“已恢复”；普通 `NORMAL` 和 `QUERY_RECOVERED` 未错误进入已恢复记录。

## 13. 查询失败验证

- [ ] 使用 fake/H2 或现场批准的非破坏性断连方式确认显示“查询失败，数据不可用”，不显示为无料、正常或已恢复；恢复后仍无料从零计时。

## 14. 日志可写

- [ ] 普通用户可写 `logs`；检测、事件和运行日志可刷新，慢日志 I/O 不阻塞监控调度。

## 15. 诊断包脱敏

- [ ] “导出诊断包”或 `生成诊断包.bat` 返回 0；ZIP 仅含六个白名单文件，敏感信息和禁止文件命中为 0。

## 16. 配置持久化

- [ ] 保存连接、点位组和八项系统设置后关闭并重启，非敏感配置正确加载；损坏配置能安全失败或回退。

## 17. 密码不持久化

- [ ] 保存、监控、切换连接、停止和重启后检查配置、日志、诊断包，均不存在密码键或密码值；重启必须重新输入密码。

## 18. 重启验证

- [ ] 正常关闭并重新运行 `启动工具.bat`，八页可切换，当前非敏感配置、默认首页和日志策略生效，旧任务未残留。

## 19. 回滚演练

- [ ] 按 `回滚说明.md` 停止新版本、恢复旧目录及匹配配置、运行旧版检查和受控检测；确认未修改业务数据库。
