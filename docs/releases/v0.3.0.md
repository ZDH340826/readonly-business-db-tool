# Release v0.3.0

Date: 2026-07-01

## Summary

Site-ready connection and point-query release for the readonly business database tool.

## Included

- Chinese connection-management labels for onsite operators.
- Default onsite PostgreSQL profile using placeholders for host and user.
- Default onsite connection shape: port `2345`, database `cms_web`, schema `public`, `sslmode=disable`.
- Site point table query against `public.tcs_map_data`.
- Stable internal aliases for point monitoring:
  - `map_data_code` -> point code
  - `pod_code` -> shelf code
  - `pod_status` -> shelf status
  - `status` -> point status
  - `ind_lock` -> lock state
  - `date_chg` -> updated time
- Local H2 test database aligned with the same `tcs_map_data` table shape.
- Documentation for local credential entry, password handling, and public-safe table mapping.

## Safety Boundary

- No SQL editor.
- No table editing UI.
- No INSERT, UPDATE, DELETE, DDL, or save-to-database workflow.
- Passwords are entered at runtime and are not saved.
- Public repository contains only sample point identifiers and table/field mapping.
- Real IPs, usernames, passwords, production point groups, and real point lists must remain local.
