# v0.5.0-rc.1 现场试运行发布说明

发布日期：2026-07-14
交付性质：Release Candidate，供 Windows 现场试运行验收，不代表已完成生产数据库联调。

## 交付范围

- 八个真实功能页面：监控总览、点位组管理、报警中心、连接管理、数据查询、数据源浏览器、日志与系统、系统设置。
- 查询失败独立语义：`QUERY_FAILED` 显示为“查询失败，数据不可用”，不触发缺料报警，不计为正常或业务恢复。
- 监控会话快照、停止/切换代际门禁、任务密码副本清零、监控与 I/O 执行器分离。
- 结构化只读查询、当前页 UTF-8 CSV 导出、元数据浏览和前 100 行预览。
- 六文件白名单脱敏诊断包、离线部署预检、内嵌 Java 运行时和三个中文脚本。
- 现场手册、验收清单、回滚说明，以及本地 H2/临时目录自动回归验证。

## 只读与数据安全边界

- 生产数据访问只允许参数化 `SELECT` 和 JDBC 元数据读取。
- PostgreSQL 连接设置 JDBC 只读、只读事务和语句超时；仍必须使用现场提供的数据库只读账号。
- 工具不提供 SQL 编辑器、任意表/字段输入、表格编辑或写入按钮。
- 密码只存在运行期内存和短生命周期 `char[]` 副本，不写入配置文件、日志、诊断包或发布包。
- 本地 H2 写入仅用于公开样例库初始化和离线场景切换，不属于生产数据库路径。

## 已完成的离线验证

- 完整 `build.ps1` 编译、自动测试、jlink runtime、包内自检、预检和诊断命令通过。
- 27 项现场场景证据映射、配置损坏回退、CSV 安全、诊断白名单和中文/空格路径通过。
- 所有数据库自动测试只使用本地 H2、fake、stub 或 mock，未访问生产数据库。

## 现场待确认

以下项目没有在本仓库或自动测试中连接真实 PostgreSQL，必须在变更窗口内由现场人员确认：

- 现场主机到 PostgreSQL 的网络、端口、SSL 模式与认证策略。
- 现场防火墙/DNS 路径和服务端 `pg_hba.conf` 规则已由 DBA 预置并生效。
- 现场只读账号能否连接目标数据库和 `public` schema。
- 当前数据库用户是否为批准的只读角色，且数据库侧拒绝写入、DDL 与授权权限。
- 只读账号对 `tcs_map_data` 固定查询与 JDBC metadata 的授权是否完整。
- 数据字段含义、状态值、锁定值、时区和更新时间是否与现场约定一致。
- 真实点位组编码、使用位/备用位关系、检测周期、持续时间和备用位阈值。
- 在受控账号/变更流程下确认写权限被数据库拒绝；本工具不会主动执行写 SQL。
- 长时间运行、断线、数据库维护、账号锁定和网络抖动下的现场表现。
- 目标 Windows 版本、终端安全软件、磁盘配额、DPI 和中文字体显示。

## 升级与回滚

RC 包采用并排解压，不覆盖旧版本。升级前备份旧目录下本地配置和日志；需要回滚时停止新版本，恢复旧目录及其配置后重新执行旧版预检。数据库不需要也不允许做任何回滚操作。详细步骤见 `回滚说明.md`。
